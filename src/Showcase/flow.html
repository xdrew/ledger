The append-only event log at the centre is the single source of truth. Commands flow in from the left and are written as events; the outbox and projections are independent catch-up subscribers that tail the same ordered log and fan derived state back out.
mode
the stages
command — a request enters the API and is dispatched on the in-process bus, carrying a correlation id.
aggregate — loaded by replaying its history, it decides and records new events; optimistic version guards the write.
event log — each event is appended at a global position. Nothing is ever updated in place.
outbox relay — tails the log from its checkpoint and publishes every event in order, at-least-once.
event bus — active transport is Postgres LISTEN/NOTIFY on channel ledger_events. NATS JetStream is only a future swap behind the EventPublisher port — not implemented. Consumers dedupe by event id (consumed_events); no external consumer is wired in yet.
projections — a separate catch-up subscription that tails the log directly and folds events into read models, exactly-once. The gap is the lag.
