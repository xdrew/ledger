> Built with the travel project's API conventions. Depends on `add-message-bus` (command dispatch)
> and accounts, transfers, idempotency, projections. New deps: `baldinof/roadrunner-bundle` +
> `symfony/runtime` (app server), `symfony/security-bundle` (firewall), `symfony/validator`.
> OpenAPI is generated by a custom reflection generator (no nelmio/swagger-php) and verified
> without an OpenAPI-validator dependency. The production image and Helm are deferred to
> add-deployment.

## 1. HTTP stack & RoadRunner (travel setup)

- [ ] 1.1 Enable the Symfony HTTP stack: `public/index.php` returning the Kernel via `autoload_runtime.php`; framework HTTP config; `config/routes/api.yaml` importing attribute routes from `src/Api` under `/api` (json format); load `App\Api\**\Action` tagged `controller.service_arguments` in `services.yaml`.
- [ ] 1.2 Add RoadRunner the travel way: `baldinof/roadrunner-bundle` + `symfony/runtime` (`APP_RUNTIME`); `.rr.yaml` + `.rr.dev.yaml` (HTTP plugin, worker pool); the `rr` binary in the dev image; a compose `http` service running `rr serve -c .rr.dev.yaml`. Verify the bundle resolves on Symfony 8.

## 2. Cross-cutting concerns

- [ ] 2.1 `ApiKeyAuthenticator` on the Symfony Security firewall: validate the API-key header; `access_control` requires it on `/api/*`; `health`/`openapi` are public. Missing/invalid → `401`.
- [ ] 2.2 Idempotency listener over mutating routes: `begin` (hash request), replay Completed, `409` InProgress, `422` Mismatch; capture the response and `complete`.
- [ ] 2.3 `ApiExceptionListener` (`KernelEvents::EXCEPTION`) mapping throwables via `match()` to status and rendering the JSON envelope (`{ message }`, plus `{ errors }` for validation). Explicit cases per the design table (ledger exceptions extend `\RuntimeException`).
- [ ] 2.4 `Request` DTOs + `symfony/validator` constraints (`#[MapRequestPayload]`); validation failure → `422` with field details via the error listener.

## 3. Controllers & endpoints (invokable `Action`, travel layout)

- [ ] 3.1 `App\Api\Accounts\Open\Action` (`POST /api/accounts`), `…\Get\Action` (`GET /api/accounts/{id}`), `…\Deposit\Action` (`POST /api/accounts/{id}/deposits`) dispatching `OpenAccount`/`DepositFunds` on the `CommandBus` / reading the balances view; each with sibling `Request`/`Response` DTOs.
- [ ] 3.2 `App\Api\Transfers\Create\Action` (`POST /api/transfers`, `#[ResponseStatus(201)]`) and `…\Get\Action` (`GET /api/transfers/{id}`).

## 4. OpenAPI & contract (custom generator, travel approach)

- [ ] 4.1 Implement `App\Infrastructure\OpenApi\OpenApiGenerator` + attributes `#[OpenApiPublic]`, `#[ResponseStatus]`, `#[Tag]`, `#[QueryParam]` (reflection over `Action`s: `#[Route]`, path params, `#[MapRequestPayload]` → schema, `JsonSerializable` return → schema; API-key security scheme) emitting OpenAPI 3.1.
